## Canon

Direction is data now. Priorities, principles, and the reasoning behind them belong where an agent can find, parse, and cite them — and current enough to be safe to obey.

## Editor's note

Every agent in the building is already making prioritization calls. The only question is whose priorities it's using.

A spec-drafting agent ranks requirements; a triage agent decides what's urgent; a grooming agent judges relevance. When direction is trapped in slideware and meeting memory, those calls fall back on the model's general instincts — fluent, plausible, and unmoored from what the org actually decided. Leaders who publish direction at a stable, dated address turn every one of those agents into an instrument of strategy instead of a generator of plausible drift.

- **Markdown over slideware.** A deck is a snapshot for a meeting; a versioned page is an interface for everything that reads it after.
- **Date everything.** An agent can't smell staleness — say when a priority was set and when it expires.
- **Write principles as rules, not posters.** "Optimize for X over Y when they conflict" is applicable; an inspirational noun is not.
