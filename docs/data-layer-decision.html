01The problem

The data layer already exists — it just wasn't decided

Content is split across JSON and Markdown today, but by accident of how it grew, not by a rule. That's fine where it landed well, and painful where it didn't.

Today — the build pipeline (content → query layer → static site)

What works — and what hurts

✓ keep

Use-case frontmatter

Structured handles on top (id, phase, techniques[], metric), three-voice prose in the body. This is the model to generalize.
✓ keep

loadContent()

Already a query layer — lookups, reverse indexes, rollups. Downstream code never touches the file layout. The de-facto database engine.
⚠ the wound

Prose inside JSON

phases.json stuffs editor's notes — with **bold** — into JSON strings. Hard to edit, no rich-text render, escaping pain. Plus: no foreign-key checks anywhere.
02The principle

One rule draws the line

The rule

Structured if you'd query it. Markdown if you'd read it.

A field goes in structured data (JSON/YAML or frontmatter) when the build or UI needs to filter, sort, join, count, validate, or link on it — ids, foreign keys, flags, tags, metrics, ordering, colors. A field goes in Markdown prose when a human writes it in sentences and it renders as rich text — goals, approaches, editor's notes, canon lines, bios. Frontmatter is the hinge that lets one file carry both.

03The proposed model

Three tiers, by what a field is for

Apply the rule and the layer resolves into three clean tiers — config, the structured skeleton, and prose entities — all feeding the same loader.

Proposed — three tiers feed one query layer
04The "no database" insight

The build is the database

Every database concept already has an equivalent here — except one. The only missing piece is the constraint layer, and it's ~40 lines.

Mapping — relational concepts → this project

Because the model maps 1:1 onto rows and columns, choosing it now is not a bet against a future database — frontmatter becomes columns, the Markdown body becomes a rich-text column, ids become keys. The migration, if it ever happens, is mechanical.
